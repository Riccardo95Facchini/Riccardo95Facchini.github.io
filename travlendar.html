Travlendar - RASD & DD

Overview

The project was developed during the Software Engineering 2 course at Politecnico di Milano during the year 2017-2018.
The task consisted in producing the requirement analysis and specification document (RASD) and the design document (DD) for a service called "Travlendar".

Description

The Travlendar system was supposed to be a helpful tool for users that never wanted to lose an appointment. The main idea was to develop a system capable of storing every appointment and notify the user by telling him/her when the best possible time to leave and with which of the selected transportation means it was better to proceed, considering factors such as weather, time, traffic, accidents and so on.
